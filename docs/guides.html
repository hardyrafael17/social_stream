Getting Started

1. Installation

Before you can use Social Stream Ninja, you'll need to install it first. There are multiple ways to install depending on your preferences:

  • Browser Extension (recommended): Install via Chrome Web Store or manual installation for the most up-to-date version.
  • Standalone App: Available for Windows, macOS, and Linux for those who prefer a desktop application.

For detailed installation instructions, please visit the Download page.

2. Basic Setup

Once installed, follow these steps to start using Social Stream Ninja:

  1. Open your streaming platform's chat in a browser (or use the pop-out chat feature if available)
  2. Click the Social Stream Ninja extension icon in your browser toolbar
  3. Click "Enable Extension" to turn it on (the icon should turn green)
  4. Open the "Dashboard Link" provided in the extension popup
  5. Open the "Overlay Link" in OBS as a browser source

Note: For the best experience, keep chat windows visible (not minimized) when using the browser extension.

3. Connecting to Different Platforms

Social Stream Ninja works with 100+ platforms. Here's how to connect to the most popular ones:

YouTube

  • Open YouTube Live chat
  • Click the "Pop out chat" button in the top-right corner
  • Ensure the Social Stream Ninja extension is enabled (green icon)

Twitch

  • Open Twitch chat for your channel
  • Click the gear icon and select "Pop out chat"
  • Ensure the Social Stream Ninja extension is enabled (green icon)

Facebook

  • Open your Facebook Live broadcast (as a viewer or publisher)
  • Ensure you're connected to WiFi or Ethernet (not mobile data)
  • Ensure the Social Stream Ninja extension is enabled (green icon)

Discord, Slack, WhatsApp, etc.

  • First, enable these platforms in the extension settings (toggle switch)
  • Open the web versions of these platforms in your browser
  • Ensure the Social Stream Ninja extension is enabled (green icon)

Customization

URL Parameters

You can customize both the dashboard and overlay by adding parameters to their URL. Some examples:

Dashboard Parameters:

&darkmode - Enables dark mode for the dashboard
&scale=2 - Doubles the size/resolution of all elements
&hidesource - Hides platform icons (YouTube, Twitch, etc.)
&compact - Removes spacing between name and message
&autoshow - Auto-features messages as they appear (about 2 per 3 seconds)

Overlay Parameters:

&showtime=20000 - Auto-hides selected messages after 20 seconds
&showsource - Shows the platform icons next to usernames
&fade - Makes featured messages fade in instead of popping up
&swipe - Makes featured messages swipe in from the left
¢er - Centers featured messages on screen

For easier URL management, try our URL Parameter Editor Tool.

CSS Customization

You can completely change the look of your chat overlay by customizing the CSS. Here are two ways to do it:

Method 1: OBS Browser Source CSS

In OBS, you can add custom CSS to the browser source:

  1. Add your overlay as a Browser Source in OBS
  2. Check the "Custom CSS" option
  3. Enter your custom CSS in the text box
body { background-color: rgba(0, 0, 0, 0); margin: 0px auto; overflow: hidden; }

:root {
    --comment-color: #090;
    --comment-bg-color: #DDD;
    --comment-border-radius: 10px;
    --comment-font-size: 30px;
    --author-border-radius: 10px;
    --author-bg-color: #FF0000;
    --author-avatar-border-color: #FF0000;
    --author-font-size: 32px;
    --author-color: blue;
    --font-family: "Arial", sans-serif;
}

.hl-name {
    padding: 2px 10px !important;
}

Method 2: URL Parameter

You can also add CSS via URL parameters:

&css=https://yourdomain.com/style.css - Link to a CSS file
&b64css=YOUR_BASE64_ENCODED_CSS - Directly embed CSS

Advanced Features

Text-to-Speech

Social Stream Ninja can read chat messages aloud using Text-to-Speech technology:

Setting Up TTS:

  1. Add &speech=en-US to your featured.html or dock.html URL
  2. For other languages, change the language code (e.g., &speech=fr-FR for French)
  3. Customize voice settings with parameters like &pitch=1&volume=1&rate=1

Premium TTS Options:

  • Kokoro TTS: Browser-based premium quality TTS (included free)
  • Google Cloud TTS: Professional-quality voices (requires Google API key)
  • ElevenLabs: Ultra-realistic voices with custom voice training
  • Speechify: Natural-sounding premium voices

Capturing TTS Audio in OBS:

For system TTS (free option), you'll need to capture the audio:

  • Use a virtual audio cable to route system audio to OBS
  • Or capture all desktop audio in OBS

Premium TTS options play directly through the browser and can be captured with "Control audio via OBS" enabled on the browser source.

Queuing and Pinning Messages

Organize how messages appear in your overlay:

Queuing Messages:

  1. Hold CTRL (or Cmd on Mac) while clicking messages in the dashboard
  2. Click the "Next in Queue" button to display them one by one
  3. Messages will appear in your overlay in the order they were queued

Pinning Messages:

  1. Hold ALT while clicking a message in the dashboard
  2. The message will stay pinned at the top of the dashboard
  3. To unpin, hold ALT and click the pinned message again

Bot Commands and Auto-Responses

Automate interactions with your audience:

Built-in Commands:

  • !joke - Tells a random geeky dad joke (can be enabled in settings)
  • hi - Welcomes anyone who says "hi" (can be enabled in settings)

Custom Commands:

Create your own commands with the custom.js file:

  1. Rename custom_sample.js to custom.js
  2. Edit the file to create your own custom commands
  3. Add the appropriate URL parameters to activate your commands

API Integration

Social Stream Ninja offers powerful API options for remote control and integration:

Server API:

Send messages to Social Stream Ninja via API:

https://io.socialstream.ninja/SESSIONID/sendChat/null/Hello - Sends "Hello" to all chats

Webhook Support:

Receive donations and events from third-party services:

  • Stripe: Set up a webhook to https://io.socialstream.ninja/SESSIONID/stripe
  • Ko-Fi: Set up a webhook to https://io.socialstream.ninja/SESSIONID/kofi
  • Buy Me A Coffee: Set up a webhook to https://io.socialstream.ninja/SESSIONID/bmac

For more detailed API documentation, visit the Commands & API page.

Troubleshooting

Solutions for common issues with Social Stream Ninja

Chat Stops When Background/Minimized

Chat messages stop appearing when browser windows are in the background or minimized. This happens because browsers throttle background tabs to save resources.

Disable Browser Throttling
  • Chrome/Edge/Brave:
    1. Open chrome://flags/#enable-throttle-display-none-and-visibility-hidden-cross-origin-iframes
    2. Set to "Disabled"
    3. Also disable chrome://flags/#calculate-native-win-occlusion
    4. Restart your browser
  • Disable Performance Throttling:
    1. Go to chrome://settings/performance
    2. Disable "Throttle Javascript timers in background"
  • Prevent Tab Discarding:
    1. Go to chrome://discards/
    2. Find chat pop-out pages
    3. Toggle off "Auto Discardable"
Alternative Approaches
  • Keep Windows Visible: Arrange windows so they're not minimized; even a small visible portion can prevent throttling.
  • Use Virtual Desktops: On Windows, use Win+Tab to create two virtual desktops, with chat windows on one and OBS on the other.
  • Try the Standalone App: The Social Stream Ninja desktop app avoids many browser throttling issues.
  • "Always on Top" Apps: Use third-party apps to pin chat windows so they remain visible but small.

Blue Bar Appears / Auto-Responder Not Working

A blue debugging bar appears or automated responses don't work when trying to use bot commands or auto-responses.

Check Chat Window Visibility
  • Make sure if using Youtube/Twitch that the pop-out chat window is open and visible
  • For some platforms, you may need to keep the main window open alongside the pop-out
Browser Compatibility
  • Auto-responder works best with Chromium-based browsers (Chrome, Edge, Brave)
  • Firefox has limited support for extension features
Remove Conflicting Apps
  • Go to chrome://apps and remove any YouTube apps that might appear
  • Check for browser extensions that might interfere with Social Stream Ninja
Chat Permissions
  • Make sure you have permission to post into the chat
  • Some platforms require you to be a subscriber or member to send chat messages
  • Try sending a manual message first to verify permissions
Hide the Blue Debug Bar
  • Start Chrome with --silent-debugger-extension-api flag to hide the blue bar
  • Windows: Create a shortcut to Chrome and add the flag to the Target field: 
    "C:\Program Files\Google\Chrome\Application\chrome.exe" --silent-debugger-extension-api
  • macOS: Run from Terminal: 
    open -a "Google Chrome" --args --silent-debugger-extension-api

Can't Export Settings or Save Files

Unable to save settings to disk or export configuration when trying to back up your Social Stream Ninja setup.

Enable File System Access API
  • In Brave:
    1. Go to brave://flags/#file-system-access-api
    2. Enable the setting
    3. Restart Brave
  • In Chrome:
    1. Go to chrome://flags/#file-system-access-api
    2. Enable the setting
    3. Restart Chrome
  • In Edge:
    1. Go to edge://flags/#file-system-access-api
    2. Enable the setting
    3. Restart Edge

Other Common Issues

Session ID Mismatch

If your extension and dock pages aren't communicating:

  • Verify that the session ID in the extension matches the one in the dock.html URL
  • To set a custom session ID: Go to Extensions settings → Social Stream Ninja → Details → Extension options
Platform-Specific Issues

If certain platforms aren't working:

  • Discord, Slack, Telegram, and WhatsApp require enabling the specific toggle in extension settings
  • Some platforms may need the main window to remain open alongside the pop-out
  • Facebook works best when connected to WiFi/Ethernet, not mobile data
Extension Stopped Working

If the extension suddenly stops capturing messages:

  • Refresh the chat page to retrigger the extension code
  • Ensure the extension is enabled (green icon, not red)
  • Check if the browser has been running for an extended period (may need restart)
macOS/Linux Browser Source Issues

If using OBS on macOS or Linux:

  • The extension may not work if hosted locally on your drive
  • Use the hosted version on socialstream.ninja instead
  • Apply custom CSS via the browser source style section in OBS
